- PWA output files under `public/` were regenerated by build and cleaned from the working tree after audit.

## Findings

- Build, typecheck, OpenAPI drift, backend lint/typecheck and backend tests pass.
- No confirmed P0 runtime/build blocker was found in this audit.
- Direct backend command names are not available in PATH, but Python module invocations work.
- Temporary migration fallback and import-boundary warnings are the main engineering risks.

## Risks

- P1: temporary fallback routes can mask FastAPI regressions and keep business logic in Next longer than intended.
- P1: frontend TS backend-core helper imports weaken the Next-as-BFF boundary.
- P1: dry smoke did not exercise real browser/API/auth/tenant flows.
- P2: Docker validation cannot run on this local machine.

## Recommended Fixes

- Add CI jobs that run `python -m ruff`, `python -m mypy`, and `python -m pytest` when direct entrypoints are unavailable.
- Run staging smoke with real services and auth tokens.
- Convert Next lint to ESLint CLI and burn down hook dependency warnings.
- Make generated PWA files either deterministic in CI or ignored from local audit runs.

## P0/P1/P2 Priority

- P0: none confirmed by build/test.
- P1: temporary fallback routes, boundary warnings, missing live smoke execution.
- P2: local PATH/tooling cleanup, Docker CLI availability, lint deprecation.
